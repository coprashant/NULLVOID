# N U L L V O I D
> Cyberpunk binary-logic runner. Java + LibGDX → TeaVM → browser.

---

## Setup (VS Code)

### Step 1 — Install extensions
Open VS Code. It will automatically suggest the required extensions
(from `.vscode/extensions.json`). Install all three:

- **Extension Pack for Java** — language support, autocomplete, errors
- **Gradle for Java** — run build tasks from sidebar
- **Live Server** — test game in browser instantly

Or search them manually in `Ctrl+Shift+X`.

---

### Step 2 — Get the gradle-wrapper.jar

The `gradlew` script needs one binary file that can't be included in the zip.
Run this **once** to download it:

**Mac/Linux:**
```bash
cd nullvoid
gradle wrapper --gradle-version 8.5
```

**Windows:**
```cmd
cd nullvoid
gradle wrapper --gradle-version 8.5
```

> **Don't have Gradle installed?**
> Download it from https://gradle.org/install/ (just for this one step).
> After this, `gradlew` handles everything — you never need global Gradle again.

---

### Step 3 — Open project in VS Code
```
File → Open Folder → select the "nullvoid" folder
```

VS Code will detect the Gradle project and start downloading LibGDX.
This takes **2–5 minutes** the first time (downloading ~50MB of dependencies).

Watch the status bar at the bottom — it shows "Java: Importing..." while it works.

---

### Step 4 — Build the web version

**Option A: Keyboard shortcut**
```
Ctrl+Shift+B
```
Runs "Build Web (TeaVM → game.js)" automatically.

**Option B: Gradle sidebar**
Open the Gradle panel (elephant icon in left sidebar) →
`nullvoid → html → Tasks → teavm → generateJs`

**Option C: Terminal**
```bash
./gradlew :html:generateJs       # Mac/Linux
.\gradlew.bat :html:generateJs   # Windows
```

The output file appears at: `html/webapp/game.js`
Build time: ~1–3 minutes.

---

### Step 5 — Test in browser

**Option A: Live Server (easiest)**
1. Right-click `html/webapp/index.html` in the VS Code explorer
2. Select "Open with Live Server"
3. Browser opens automatically at `http://localhost:5500`

**Option B: Python server**
```bash
cd html/webapp
python3 -m http.server 8080
# open http://localhost:8080
```

**Option C: VS Code task**
```
Ctrl+Shift+P → "Run Task" → "Serve Game Locally (port 8080)"
```

> ⚠️ Don't double-click index.html to open it — browsers block local JS files.
> Always use a server.

## Project Structure

```
nullvoid/
│
├── .vscode/
│   ├── settings.json     ← VS Code project config (auto-detects Java/Gradle)
│   ├── tasks.json        ← Ctrl+Shift+B = build web version
│   └── extensions.json   ← Suggests extensions when project is opened
│
├── core/                 ← ALL YOUR GAME CODE LIVES HERE
│   └── src/main/java/com/nullvoid/
│       ├── NullVoid.java       Main class — game loop (create/render/dispose)
│       ├── GameWorld.java      Game logic — packet, gates, bits, score
│       ├── GameUI.java         Menus and screen overlays
│       └── InputHandler.java   Keyboard / touch input
│
├── html/                 ← Web export (don't write game logic here)
│   └── src/main/java/com/nullvoid/
│       └── HtmlLauncher.java   TeaVM entry point
│   └── webapp/
│       ├── index.html          The web page (embed this in your portfolio)
│       └── game.js             ← Generated by TeaVM after build
│
├── assets/               ← Images, sounds go here (empty for now)
├── build.gradle          ← Root build config
├── settings.gradle       ← Module list
├── gradlew               ← Build script (Mac/Linux)
└── gradlew.bat           ← Build script (Windows)
```

## How to Play

| Key | Action |
|-----|--------|
| ↑ or W | Move packet up one lane |
| ↓ or S | Move packet down one lane |
| SPACE | Start / return to menu |

**Goal:** A Firewall Gate approaches with a decimal number.
Toggle bit pads (8, 4, 2, 1) so your total matches.

`13 = 8 + 4 + 1` → activate lanes 8, 4, 1 ✓

## Common Issues

| Problem | Fix |
|---------|-----|
| `gradlew: Permission denied` | Run `chmod +x gradlew` |
| `Could not resolve gdx` | Check internet connection, run `./gradlew dependencies` |
| `gradle-wrapper.jar not found` | Run `gradle wrapper --gradle-version 8.5` once |
| Canvas is blank | Use a local server, not file:// |
| Build takes forever | Normal first time — LibGDX is ~50MB download |
